## Why Flux

Flux connects one or more Telegram accounts over MTProto and turns each one into an **instance** you manage over HTTP — read chats, send messages and media, and receive every update as realtime events or signed webhooks. Built on **NestJS 11 + Prisma 7 (PostgreSQL) + Redis**, with a **Vue 3 dashboard** to drive it all visually.

<CardGrid stagger>
  <Card title="Accounts as instances" icon="telegram">
    Connect via **QR code** or **phone (OTP) + 2FA**. Sessions persist and reconnect automatically across restarts.
  </Card>
  <Card title="Clean REST API" icon="open-book">
    Chats, history, messages and media over HTTP. Interactive OpenAPI reference served at `/docs` (Scalar).
  </Card>
  <Card title="Realtime + webhooks" icon="rocket">
    New/edited/deleted messages, read receipts, reactions and session status — over **SSE** and durable, **HMAC-signed webhooks**.
  </Card>
  <Card title="Zero-config boot" icon="setting">
    No required env vars. Secrets are generated and persisted; an admin is seeded on first boot. `docker compose up` and go.
  </Card>
</CardGrid>
