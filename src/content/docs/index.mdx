Flux turns one or more Telegram accounts into **instances** you drive over HTTP — authenticate, read and send messages, and receive every update as realtime events or signed webhooks. This guide walks through everything you can do, feature by feature.

## What you can do

- **Run many accounts** — each Telegram number is an [instance](/flux-docs/instances/) with its own session, operated independently from one gateway.
- **Read & send messages** — list chats, page through history, send text and media (photos, videos, documents up to 50 MB) over plain HTTP.
- **React in realtime** — new messages, edits, deletes, read receipts, reactions and status changes stream live over [SSE](/flux-docs/events/), no polling.
- **Get signed webhooks** — subscribe your endpoint to any subset of events; deliveries are HMAC-signed, stored in a Postgres outbox and [retried with backoff](/flux-docs/webhooks/).
- **Control access** — dashboard [accounts](/flux-docs/accounts/) with admin / operator / viewer roles, Argon2id passwords, encrypted sessions and SSRF-guarded webhook URLs.

## See it

A Vue dashboard ships with the gateway — manage instances, sessions, webhooks and users without touching the API:

<Image src={overview} alt="Flux dashboard Overview: uptime, instance and webhook counts, and service health" />

Prefer code? The same things are one HTTP call away:

```bash
# Send a message from an authorized instance
curl -X POST \
  http://localhost:3000/telegram/instances/<id>/chats/<chatId>/messages \
  -H 'Authorization: Bearer <JWT>' -H 'x-api-key: <API_KEY>' \
  -H 'Content-Type: application/json' \
  -d '{"text":"Hello from Flux"}'
```

## Up and running in one command

The published all-in-one image bundles the API, PostgreSQL and Redis — no setup, no config:

```bash
docker run -d -p 3000:3000 -v flux_data:/data pedrooaj/flux-api
```

Then open the dashboard at `http://localhost:3000/dashboard`, or the interactive
API docs at `/docs`. Full walkthrough in [Getting started](/flux-docs/getting-started/);
the request-to-endpoint flow is in [How it works](/flux-docs/how-it-works/).

Built on **NestJS 11 + Prisma 7 (PostgreSQL) + Redis**, with a **Vue 3** dashboard.
