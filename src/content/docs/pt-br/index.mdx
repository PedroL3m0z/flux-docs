A Flux transforma uma ou mais contas do Telegram em **instâncias** que você controla por HTTP — autentique, leia e envie mensagens, e receba cada atualização como eventos em tempo real ou webhooks assinados. Este guia percorre tudo o que dá para fazer, recurso por recurso.

## Por que a Flux

<CardGrid>
  <Card title="Open source" icon="github">
    Apache-2.0, código no GitHub. Leia, faça fork, rode — sem caixa-preta, sem aprisionamento de fornecedor.
  </Card>
  <Card title="Self-hosted" icon="setting">
    Suas contas, seu servidor, seus dados. Um container com Postgres + Redis embutidos, ou escale com o Compose.
  </Card>
  <Card title="REST API" icon="list-format">
    Cada ação do Telegram é um endpoint HTTP simples, com spec OpenAPI e docs interativas (Scalar) em `/docs`.
  </Card>
  <Card title="Simples e fácil" icon="rocket">
    Boot sem configuração — `docker run` e pronto. Segredos e o usuário admin são gerados no primeiro start.
  </Card>
  <Card title="Webhooks" icon="random">
    Receba cada evento no seu endpoint — assinado (HMAC), com retentativas e backoff, e um log completo de entregas.
  </Card>
</CardGrid>

## Veja na prática

Um dashboard em Vue acompanha o gateway — gerencie instâncias, sessões, webhooks e usuários sem tocar na API:

<Image src={overview} alt="Dashboard da Flux (Overview): uptime, contagem de instâncias e webhooks, e saúde dos serviços" />

Prefere código? As mesmas coisas estão a uma chamada HTTP de distância:

```bash
# Enviar uma mensagem por uma instância autorizada
curl -X POST \
  http://localhost:3000/telegram/instances/<id>/chats/<chatId>/messages \
  -H 'Authorization: Bearer <JWT>' -H 'x-api-key: <API_KEY>' \
  -H 'Content-Type: application/json' \
  -d '{"text":"Olá da Flux"}'
```
